# 需求文档

## 1. 项目背景
组织《三角洲行动》《PUBG》《CF》《CSGO》的游戏比赛，需要一套用于赛事反作弊与合规检查的工具体系，重点降低选手使用外挂、宏、违规插件、异常外设、可疑驱动等行为带来的赛事风险。

## 2. 总目标
构建一套 Windows 赛事反作弊原型系统，覆盖：
- 赛前检测选手电脑是否安全
- 赛中持续监控
- 赛后生成证据与裁判复核材料

## 3. 适用场景
- 主要场景：线上远程比赛
- 终端环境：选手自有 Windows 电脑
- 使用方式：赛前启动检测，赛中持续运行，赛后导出报告

## 4. 已确认需求
### 4.1 结果形式
- 需要方案清单
- 需要实际开发一款工具

### 4.2 检测范围
需覆盖以下方向：
- 软件进程检查
- 外挂辅助检查
- 硬件外设检查
- 系统内核检查
- 裁判报告导出

### 4.3 检测时段
- 赛前检测是否安全
- 安全后进入赛中持续监控
- 持续到赛后

### 4.4 风险处理方式
采用分级处理：
- 低风险：提醒并上报
- 中风险：限制进入比赛 / 要求复检 / 裁判复核
- 高风险：自动锁定比赛资格并通知裁判

### 4.5 客户端权限要求
必须采用方案 2：
- 用户态客户端 + 内核驱动 + 云端后台
- 允许使用内核级客户端能力

## 5. 已确认架构方案
采用以下总体架构：
- 选手端桌面客户端
- 用户态扫描代理
- Windows 内核驱动
- 云端规则服务 / 风险评分服务
- 裁判后台
- 审计与证据存储

## 6. 核心流程
1. 选手登录客户端
2. 客户端与驱动完成自检
3. 拉取规则并执行赛前全量扫描
4. 本地初判 + 云端复判
5. 生成安全评分与分级结果
6. 通过后进入赛中持续巡检
7. 赛中持续上报状态、异常与证据
8. 赛后生成证据包与裁判报告

## 7. 关键能力要求
### 7.1 赛前检测
- 已安装软件检查
- 进程、服务、驱动检查
- 启动项检查
- 覆盖层、宏、脚本、调试器、注入器、抓包器检查
- USB/HID/异常外设检查
- 虚拟机/内核异常检查

### 7.2 赛中持续监控
- 心跳上报
- 周期巡检
- 事件触发巡检
- 风险分更新
- 异常自动告警

### 7.3 赛后审计
- 证据打包
- 日志归档
- 时间线整理
- 裁判复核报告导出

## 8. 数据与证据要求
每条风险结论应尽量绑定证据，包括但不限于：
- 命中规则 ID
- 风险级别
- 事件摘要
- 文件/日志哈希
- 采集时间
- 本地扫描结果
- 裁判复核说明

## 9. 技术方向要求
当前实现计划采用：
- .NET 8
- WPF / Windows Desktop
- ASP.NET Core
- PostgreSQL
- Redis
- SignalR / WebSocket
- KMDF/C 内核驱动
- xUnit
- Playwright
