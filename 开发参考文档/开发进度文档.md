## 2. 对话需求总览
### 2.1 最初目标
用户希望为《三角洲行动》《PUBG》《CF》《CSGO》组织比赛，需要一套反作弊检测工具，用于检查选手电脑是否存在违规插件、软件、硬件等。

### 2.2 已确认的关键需求
- 既要方案，也要实际开发工具
- 主要场景为线上远程比赛
- 工具需要覆盖：
  - 软件进程
  - 外挂辅助
  - 硬件外设
  - 系统内核
  - 裁判报告
- 赛前检查是否安全
- 若安全，赛中持续监控直到赛后
- 风险处理方式为分级处理
- 必须采用方案 2：用户态 + 内核级客户端 + 云端后台
- 允许内核级检测能力
- 根目录需要有 `需求文档.md` 与 `开发进度文档.md`
- 项目主要目录下需要各自维护 `说明.md`
- 清理聊天过程中对项目无用的构建产物和临时文件

### 2.3 用户后续新增要求
- 部署 .NET 8 SDK 后继续开发
- 将项目上传到 GitHub 仓库 `https://github.com/jiangchengbei/mz.git`
- 在根目录生成总结性文档，便于后续继续开发
- 对项目目录进行整理并补说明文档
